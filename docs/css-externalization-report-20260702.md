# CSS外部化レポート（2026-07-02）

全157ページ中156ページのインライン`<style>`を [assets/site.css](../assets/site.css) 1本に統合した。
404.htmlのみヘッダー/フッターを持たない独立最小ページのためインラインのまま（269B）。

## 1. インラインCSSの分析結果（21バリエーション）

| 系統 | シート | ページ数 | 性質 |
|---|---|---|---|
| 新世代詳細（正として採用） | 7091B | 48 | 会社導線・@media print・タイムライン等を含む最新の完成形 |
| 旧世代詳細 | 4476B / 4649B / 4618B | 49+14+10 | 共有セレクタ方式の旧アーキテクチャ。ボタン角丸10px・フォント13.5px等 |
| 新世代の単発派生 | 6-8KB ×13種 | 各1〜6 | 新世代コア＋ページ固有部品（緊急通報ボタン・時系列タイムライン・カード色・緑CTA・ヒーロー色調） |
| ハブ | 6984B / 7514B | 13+1 | トップ・カテゴリ用の**正当に異なるデザイン**（.wrap 980px・btn 44px・検索UI等） |
| 404 | 269B | 1 | 独立ページ（対象外） |

判別結果:
- **ハブ vs 詳細 = 正当なデザイン差**。`body.hub` / `body.hub.home`（トップ）スコープで完全保存した。
- **詳細ページ内の差 = 世代差**（旧世代73ページ / 新世代66ページ / 単発派生）。単なる取り残しではなく新旧2大世代が拮抗していたため、「最多数派＝最新版」の前提は成立せず（例: ボタンは旧世代が73ページで多数派だが設計憲章v1.1由来の会社導線・印刷対応等を持つのは新世代）。**設計憲章を実装した新世代コア（7091B・48ページ）を「最新世代の正」として採用**した。
- **ページ固有の意図的デザインは保存**: 緊急通報ボタン(119/#7119/#8000)、引っ越しタイムライン、real-cardのページ別アクセント色16種、住まい相談の緑CTA(#06c755)。

## 2. site.css の構成

```
共通(:root変数の和集合 / * / html,body)
詳細ページ  body:not(.hub) …  ← 新世代コア + 固有部品 + 旧世代の.a〜.d色
カテゴリハブ body.hub …       ← ハブシートそのまま(enshu-footerはparts/footer.htmlが供給)
トップ      body.home …       ← 検索UI・quick-grid 2列のみ(他はbody.hubを共有)
```

- ハブ13ページ=`<body class="hub">`、トップ=`<body class="hub home">` を付与（描画に影響しないマークアップ変更）。
- 同名クラスの色衝突はHTML側のクラス改名で解消（見た目は完全維持）:
  - want-to-live: `real-card home`(#0d8f6f)→`zone` / `btn-main`(緑)→`btn-green`
  - about-iwata: `btn-main`(緑)→`btn-green`
  - developmental-support: `real-card talk`(#0d8f6f)→`zone`（talkは#d85b7fでsingle-parent-supportが使用）

## 4. 検証方法と結果

- ヘッドレスブラウザは本環境で利用不可のためCSS実効diffで検証。
- 検証器: 各ページのHTMLをスタックパースして全要素の祖先チェーンを構築し、旧インラインCSSとsite.cssのカスケード勝者（メディア・擬似クラス・プロパティ単位、詳細度＋出現順）を全156ページで比較。
- 残差分は上記リスト＋等価項目（border-topショートハンド⇔border-top-color、inline styleが勝つ箇所、既定値と同値のflex-wrap等）のみであることを確認。
- ハブ・トップは全プロパティ一致（=1pxも変わらない）。

**目視確認の推奨ポイント**
1. トップ・カテゴリハブ（変化ゼロのはず）
2. 旧世代代表: /life/work-life/start-business/（ボタンの角丸・文字サイズ）
3. 新世代代表: /life/emergency/disaster-certificates-procedures/（変化はフッターリンク色のみのはず）
4. 固有部品: /life/emergency/emergency-medical/（119ボタン）、/life/living-soon/moving-decided/（タイムライン）、/life/living-soon/want-to-live/（緑CTA・カード色）
5. モバイル表示: 任意の旧世代ページのヒーロー高さ・フィードバック1列

## 5. 今後の運用

- CSSを変更したら `parts/head-css.html` の `?v=` を更新し `python scripts/inject_parts.py` を実行（全ページのキャッシュバスターが一括更新される）。
- 新規ページは `<link rel="stylesheet" href="/assets/site.css?v=...">` をPARTマーカー付きで入れる（インライン`<style>`は追加しない）。
