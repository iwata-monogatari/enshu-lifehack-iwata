# トップページ3層導線ルール（2026-07-04確定・9市町展開の標準仕様）

出典: `01作業手順書\20260704\iwata-lifehack-top-kaizen-shijisho.md` No.3

## 3層の役割分離

- **クイック入口（上段・10件固定）**: 手続き名で探す人向け。名詞ベースの頻出手続き。
- **カテゴリ（中段・13件固定）**: サイトの全体地図。構成変更しない。
- **困りごと（下段・上限12件、目安10件前後）**: 状況文のみ。手続き名を知らない人向けの逆引き導線。

## 困りごとの選定基準

- **残す**: 手続き名に変換しにくい状況文（例：家族が亡くなった、親の介護が心配、空き家をどうするか）。
- **削る**: クイック入口・カテゴリと同義の名詞導線の言い換え（例：住民票がほしい、ごみの日が分からない）。
- 削った項目は削除せず、対応するカテゴリページの冒頭（heroセクション直後）に `class="quick-situations"` セクション（見出し「こんなときはこのページ」、中身は既存の `.trouble-grid`/`.trouble-card` を流用）として移設し、リンク資産を保持する。

## 磐田版での適用結果（参考値）

- 困りごと20件→9件に削減。削った11件は次の7カテゴリページの冒頭へ分散移設。
  - start-living（3件）、end-of-life（1件）、living-soon（1件）、family-grow（2件）、parents-care（2件）、emergency（1件）、moving-out（1件）
- 横展開時も同じ配分パターン（多いカテゴリ=暮らし始めた/親のこと/家族が増える）になりやすい。

## 横展開時の注意

- `data/topics_master.json` にはトップページの困りごとリストは含まれない（index.html に直書き）。カテゴリごとの重複整理は各市町のトップページ・カテゴリページを個別に確認すること。
- 検索インデックス（`search-index.json`）は困りごとリンクの増減と無関係（`topics_master.json` + 各ページの `<p class="lead">` から生成されるため）。トップページ改修だけなら `build-search-index.mjs` の再実行は不要。
